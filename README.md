# What to Eat - 今天吃什么转盘

> 每天中午最头疼的问题：吃什么？让转盘帮你决定！

一个轻量的午餐选择转盘应用，纯前端实现，零依赖，打开即用。

## 功能特性

### 转盘抽奖
- 支持「转菜品 / 转店铺」两种模式切换
  - **转菜品**：转盘为所选店铺的全部菜品，抽中显示菜名、店铺和参考价
  - **转店铺**：转盘为所选店铺本身，抽中显示店铺名（先选店再决定吃哪道）
- 精美的彩色扇形转盘，支持任意数量的条目
- 流畅的旋转动画（5 秒缓动停止，模拟真实转盘惯性）
- 每次旋转 8~12 圈，保证参与感
- 转盘上显示菜名、参考价格、所属店铺（根据数量自动隐藏避免拥挤）

### 店铺筛选
- 左侧面板展示所有店铺，支持复选框多选
- 一键「全选 / 全不选」
- 可只选一家店铺，只转该店菜品
- 也可选多家或全部店铺混合转盘
- 每个店铺标签显示菜品数量和平均价格

### 价格展示
- 转盘扇形上显示参考价格（红色标注）
- 选中结果下方显示「参考价 ￥xx」
- 菜单预览区每道菜旁显示价格
- 底部统计总菜品数和总金额

### 菜单编辑器
- 点击「编辑菜单」按钮打开弹窗编辑器
- 可自由增删店铺、增删菜品、修改名称和价格
- 支持店铺和菜品的上下排序（↑↓按钮）
- 编辑后自动保存到浏览器 `localStorage`，刷新不丢失
- 下次打开优先读取本地缓存，无缓存时加载默认 `menu.json`
- 支持「恢复默认」一键重置

### 抽签记录
- 右侧面板记录最近 50 次抽签结果
- 每条记录带「菜品 / 店铺」类型徽章，区分抽的是菜还是店
- 每条记录包含：序号、类型、菜名/店铺名、价格、所属店铺、时间
- 时间智能显示：`今天 12:30` / `昨天 18:05` / `6/28 09:15`
- 记录同样缓存在 `localStorage`，刷新不丢失
- 支持一键清空

### 响应式设计
- 桌面端三栏布局（店铺选择 | 转盘 | 抽签记录）
- 移动端自动切换为单栏布局
- 转盘尺寸自适应
- 小屏手机编辑器自适应，按钮不会溢出

## 快速开始

### 方式一：直接打开

```bash
# 克隆仓库
git clone https://github.com/siyangwin/what-to-eat.git
cd what-to-eat

# 用浏览器打开 index.html（需要本地服务器才能加载 menu.json）
# 推荐使用 VS Code Live Server 或 Python 内置服务器：
python -m http.server 8080
# 然后访问 http://localhost:8080
```

### 方式二：纯静态文件

如果只是第一次加载，`index.html` 可以直接双击打开。之后菜单会缓存在 `localStorage`，不再依赖 `menu.json`。

## 自定义菜单

### 方法一：通过编辑器

1. 打开页面，点击左上角「编辑菜单」按钮
2. 在弹窗中增删店铺和菜品，支持上下排序
3. 填写菜名和参考价格
4. 点击「保存」

### 方法二：修改 menu.json

编辑 `menu.json` 文件，格式如下：

```json
{
  "shops": [
    {
      "name": "店铺名称",
      "items": [
        {"name": "菜品名称", "price": 25},
        {"name": "另一道菜", "price": 18}
      ]
    }
  ]
}
```

> 提示：修改 `menu.json` 后，需要清除浏览器 localStorage 中的 `lunch_wheel_menu` 才能加载新内容；或通过编辑器点击「恢复默认」。

## 技术栈

- **HTML5 Canvas** - 转盘绘制与动画
- **CSS3** - 渐变背景、Flex 布局、响应式设计、弹窗动画
- **Vanilla JavaScript** - 零框架，原生实现全部逻辑
- **localStorage** - 菜单与抽签记录本地持久化
